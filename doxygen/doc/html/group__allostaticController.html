Description

On top of the homeostasis module, the allostatic controller monitors all needs in homeostasis and triggers corresponding behaviors in function of which need requires to be addressed sooner. We call this pre-defined sensory-motor association a drive. This module creates needs in homeostasis from a config file or RPC command, and provides an interface to freeze them and change their parameters depending on their internal values. This module is strongly dependent on sensationManager, homeostasis and behaviorManager and is the core of the reactiveLayer. See the default.ini config file to see an example on how to define a new drive.

Parameters

  • – name : name of the module
  • – period : change the update rate of this module

Input Ports

  • /AllostaticController/rpc [Bottle] [default carrier:tcp]: Response port. Supported commands:
    • help
    • ['manual'] ['on'/'off'] : sets the allostaticController in manual mode. It the drives needs are frozen and behaviors can be triggered through rpc commands or a gui
  • /AllostaticController/toHomeoRPC:o [Bottle] [default carrier:tcp]: Communication port to create and manage homeostatic needs.
  • /AllostaticController/toBehaviorManager:o [Bottle] [default carrier:tcp]: Communication port to trigger behaviors.
  • /AllostaticController/driveName/min:o [Bottle] [default carrier:tcp]: Port to quickly read when a need crosses its lower boundary.
  • /AllostaticController/driveName/max:o [Bottle] [default carrier:tcp]: Port to quickly read when a need crosses its upper boundary.
